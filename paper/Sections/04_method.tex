\section{Method: PCU-Select}
\label{sec:method}

PCU-Select learns the surrogate in Eq.~\eqref{eq:surrogate} and uses it to select a budgeted subset for a target PEFT and task. Figure~\ref{fig:architecture} summarizes the pipeline. The offline stage extracts reusable sample and task signatures, builds low- and high-fidelity utility labels, and trains the scorer. The online stage encodes a target $(p^{*},t^{*})$, checks whether $p^{*}$ is outside the training configuration distribution, scores the pool, and allocates the budget across semantic clusters.

\subsection{Intervention Sites}

PCU-Select compares PEFT methods through module-output sites rather than raw parameter tensors. For a decoder-only Transformer, it selects eight evenly spaced layers and hooks three outputs in each layer: attention output, MLP output, and post-residual block output. The resulting set $\Omega$ has 24 sites. Module outputs are stable across backbones with fused or split projection implementations, and they keep the cache size manageable.

Each PEFT configuration maps to a site mask and an operator type. Attention LoRA and (IA)$^3$-attention touch attention-output sites; MLP LoRA and (IA)$^3$-FFN touch MLP-output sites; adapters touch block-residual sites; prefix tuning touches attention-output sites; and BitFit is represented as a bias shift. PCU-Select assigns each site a normalized PEFT weight
\begin{equation}
  \begin{aligned}
  \alpha_p^{\omega}
  &=\mathrm{mask}(p,\omega)\rho_{\mathrm{op}}c(p,\omega),\\
  c(p,\omega)
  &=\tanh\!\left(\eta\log(1+\mathrm{cap}(p,\omega)/d_{\mathrm{model}})\right),\\
  \tilde{\alpha}_p^{\omega}
  &=\alpha_p^{\omega}/\max(\epsilon,\sum_{\omega'}\alpha_p^{\omega'}).
  \end{aligned}
  \label{eq:alpha}
\end{equation}
This weight is zero for untouched sites and increases with the target PEFT's capacity at touched sites.

\subsection{Representations and Utility Labels}

A sample representation $z_x=[e_x;d_x;a_x]$ concatenates a 768-dimensional semantic embedding, 16 difficulty and format statistics, and a 64-dimensional activation signature. A task sketch $V_t$ is encoded by pooling the same sample representation over 32 sketch examples and by averaging their site gradient signatures. A PEFT code $z_p=[m_p;c_p;r_p]$ contains a $24\times4$ site/operator mask, a 16-dimensional capacity vector, and a 16-dimensional training-recipe vector.

Low-fidelity utility aligns each sample with the target sketch on the sites trained by the target PEFT:
\begin{equation}
  u^{\mathrm{lo}}(x,p,t)=\sum_{\omega\in\Omega}\tilde{\alpha}_p^{\omega}
  \cos\!\left(g_x^{\omega},g_t^{\omega}\right).
  \label{eq:ulo}
\end{equation}
Here $g_x^{\omega}$ is a unit-normalized random projection of the response-token loss gradient at site $\omega$, and $g_t^{\omega}$ is the sketch mean. The proxy is cheap after sample signatures are cached, so it labels many $(x,p,t)$ triples.

High-fidelity utility corrects the proxy near the selection boundary. For a sampled triple, PCU-Select performs short PEFT updates from a base anchor and a warm anchor, then measures sketch-loss reduction:
\begin{equation}
  \Delta(x,p,t,a,h)=L_{V_t}(\theta_a)-
  L_{V_t}\!\left(\mathrm{Adapt}^{h}_{p}(x;\theta_a)\right).
  \label{eq:delta}
\end{equation}
Rank-normalized reductions from horizons $h\in\{1,4\}$ and both anchors define $u^{\mathrm{hi}}$. The high-fidelity budget is 10K triples, split across coverage sampling, uncertainty sampling, and boundary sampling.

\subsection{Scorer and Selection}

The scorer predicts $(\hat{\mu},\hat{\sigma})=s_{\phi}(z_x,z_p,z_t)$. Three encoders map $z_x$, $z_p$, and $z_t$ into hidden vectors. The PEFT and task codes modulate the sample code through FiLM, while a bilinear term models sample-PEFT interaction. Training combines high-fidelity ranking and regression losses, low-fidelity proxy distillation, and a heteroscedastic uncertainty loss. Stage A pretrains on the proxy for broad conditional structure; stage B enables all losses and selects checkpoints on held-out PEFT/task pairs.

At application time, PCU-Select ranks examples by a conservative score
\begin{equation}
  q(x)=\hat{\mu}(x,p^{*},t^{*})-\lambda\hat{\sigma}(x,p^{*},t^{*}),\qquad \lambda=0.2.
  \label{eq:qscore}
\end{equation}
Pure top-$k$ ranking can over-select near-duplicates, so PCU-Select clusters examples in semantic space and allocates budget across clusters:
\begin{equation}
  b_k=\mathrm{round}\!\left(
  B\frac{(v_k^{+})^{\alpha}|C_k|^{1-\alpha}}
  {\sum_j(v_j^{+})^{\alpha}|C_j|^{1-\alpha}}\right),\qquad \alpha=0.6.
  \label{eq:quota}
\end{equation}
The final subset takes the top-$b_k$ examples by $q$ inside each cluster. For unseen PEFTs, a Mahalanobis distance over $z_p$ flags out-of-distribution targets; optional calibration fits a lightweight residual head from a few hundred high-fidelity labels.
