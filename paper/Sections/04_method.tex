\section{Method: PCU-Select}
\label{sec:method}

PCU-Select learns the surrogate in Eq.~\eqref{eq:surrogate} and uses it to select a budgeted subset for a target PEFT and task. Figure~\ref{fig:architecture} summarizes the pipeline. The offline stage extracts reusable sample and task signatures, builds low- and high-fidelity utility labels, and trains the scorer. The online stage encodes a target $(p^{*},t^{*})$, assigns its preregistered structural support tier, scores the pool, and allocates the budget across semantic clusters. Algorithms~\ref{alg:offline} and~\ref{alg:online} in the appendix state both stages.

\subsection{Intervention Sites}

PCU-Select compares PEFT methods through module-output sites rather than parameter tensors. It hooks attention, MLP, and post-residual outputs at eight evenly spaced Transformer layers, giving $|\Omega|=24$. By the chain rule, a PEFT parameter gradient factors through the gradient at the corresponding module output; these shared coordinates also avoid fused-versus-split projection shapes. The post-residual coordinate represents adapters alongside LoRA and (IA)$^3$. Appendix Table~\ref{tab:site-ablation} varies layer coverage, site type, and projection dimension: the default is within 0.08 points of larger site spaces while costing up to $4\times$ less.

Each PEFT configuration maps to a site mask and an operator type. Attention LoRA and (IA)$^3$-attention touch attention-output sites; MLP LoRA and (IA)$^3$-FFN touch MLP-output sites; adapters touch block-residual sites; prefix tuning touches attention-output sites; and BitFit is represented as a bias shift. PCU-Select assigns each site a normalized PEFT weight
\begin{equation}
  \begin{aligned}
  w_p^{\omega}
  &=\mathrm{mask}(p,\omega)c(p,\omega),\\
  c(p,\omega)
  &=\tanh\!\left(\eta\log(1+\mathrm{cap}(p,\omega)/d_{\mathrm{model}})\right),\\
  \tilde{w}_p^{\omega}
  &=w_p^{\omega}/\max(\epsilon,\sum_{\omega'}w_p^{\omega'}).
  \end{aligned}
  \label{eq:site-weight}
\end{equation}
This weight is zero for untouched sites and increases with capacity at touched sites. The mapping in Appendix Table~\ref{tab:capacity} places LoRA rank, adapter width, (IA)$^3$ vector count, prefix length, and BitFit bias count on a trainable-parameter scale. The $\tanh$ prevents high-capacity configurations from dominating. Operator type is encoded separately in $z_p$; a family-wide scalar would cancel under the normalization in Eq.~\eqref{eq:site-weight}.

\subsection{Representations and Utility Labels}

A sample representation $z_x=[e_x;d_x;a_x]\in\mathbb{R}^{848}$ concatenates a 768-dimensional semantic embedding, 16 difficulty and format statistics, and a 64-dimensional activation signature. The semantic vector is the frozen \texttt{sentence-transformers/all-mpnet-base-v2} joint instruction-response embedding~\cite{song2020mpnet,reimers2019sentencebert}; we do not train this encoder on the candidate pool or target benchmarks. The difficulty vector contains instruction length, response length, total length, response/instruction ratio, seven response-LM statistics (mean/std/max loss, perplexity, average log-probability, mean/max entropy), and five format flags (reasoning marker, code marker, question marker, English, Chinese). The activation signature mean-pools response-token hidden states at the eight sampled layers and projects them to 64 dimensions.

A task sketch $V_t$ is encoded to $z_t\in\mathbb{R}^{848}$ by pooling the same sample representation over 32 sketch examples and averaging their site gradient signatures. The reported PEFT code $z_p=[m_p;c_p;r_p]\in\mathbb{R}^{128}$ concatenates a $24\times4$ site/operator mask ($m_p$, 96 dims), a 16-dimensional capacity vector ($c_p$), and a 16-dimensional training-recipe vector ($r_p$). The mask channels encode active sites, additive updates, multiplicative scaling, and prefix-style control. Capacity slots distinguish LoRA, adapters, (IA)$^3$, prefix-style methods, and BitFit; recipe features encode module count, placement, and optimizer settings. At the 24-site granularity, q/v and q/k/v/o LoRA both touch attention-output sites, so capacity and recipe features retain their projection-level distinction.

Prefix, P-Tuning, and BitFit use the same deterministic site, operator, capacity, and recipe rules as the seen families. The reported model relies exclusively on these structural features, without a learned family lookup or operator centroid. Zero-shot behavior therefore measures structural transfer; where native short-horizon labels exist, calibration fits the residual head described below.

Low-fidelity utility aligns each sample with the target sketch on the sites trained by the target PEFT:
\begin{equation}
  u^{\mathrm{lo}}(x,p,t)=\sum_{\omega\in\Omega}\tilde{w}_p^{\omega}
  \cos\!\left(g_x^{\omega},g_t^{\omega}\right).
  \label{eq:ulo}
\end{equation}
Here $g_x^{\omega}$ is a unit-normalized Johnson--Lindenstrauss random projection~\cite{johnson1984jl} of the response-token loss gradient at site $\omega$, and $g_t^{\omega}$ is the sketch mean. All task sketches use the same response-token cross-entropy for signatures; task-native metrics such as option likelihood, Pass@1, and F1 are used only for downstream evaluation. The cosine proxy can be negative, so PCU-Select rank-normalizes $u^{\mathrm{lo}}$ within each $(p,t)$ bucket before proxy distillation and ranking losses. The proxy is cheap after sample signatures are cached, so it labels many $(x,p,t)$ triples.

High-fidelity utility corrects the proxy near the selection boundary. For a sampled triple, PCU-Select performs short PEFT updates from a base anchor and a warm anchor, then measures sketch-loss reduction:
\begin{equation}
  \Delta(x,p,t,a,h)=L_{V_t}(\theta_a)-
  L_{V_t}\!\left(\mathrm{Adapt}^{h}_{p}(x;\theta_a)\right).
  \label{eq:delta}
\end{equation}
Rank-normalized reductions from horizons $h\in\{1,4\}$ and both anchors define $u^{\mathrm{hi}}$. The high-fidelity budget is 10K triples, split across coverage sampling, uncertainty sampling, and boundary sampling.
Each high-fidelity update uses the sampled example plus seven in-bucket fillers; labels therefore estimate a stable mini-batch marginal, not an isolated single-example gradient. Coverage and uncertainty labels are generated first, a provisional scorer is trained, and boundary sampling then targets scorer-vs-label rank disagreements from that provisional pass. Rank normalization is always within the same PEFT, task, anchor, and horizon bucket.

\subsection{Scorer and Selection}

The scorer predicts $(\hat{\mu},\hat{\sigma})=s_{\phi}(z_x,z_p,z_t)$. Three encoders map $z_x$, $z_p$, and $z_t$ into hidden vectors. The PEFT and task codes modulate the sample code through FiLM~\cite{perez2018film}, while a bilinear term models sample-PEFT interaction. Training combines high-fidelity ranking and regression losses, low-fidelity proxy distillation, and a heteroscedastic uncertainty loss. Stage A pretrains on the proxy for broad conditional structure; stage B enables all losses and selects checkpoints on held-out PEFT/task pairs. We fix the sketch size, label allocation, anchors, horizons, and scalar selection hyperparameters on these held-out pairs before downstream evaluation.

At application time, PCU-Select ranks examples by a conservative score
\begin{equation}
  q(x)=\hat{\mu}(x,p^{*},t^{*})-\lambda\hat{\sigma}(x,p^{*},t^{*}),\qquad \lambda=0.2.
  \label{eq:qscore}
\end{equation}
Pure top-$k$ ranking can over-select near-duplicates, so PCU-Select clusters examples in semantic space and allocates budget across clusters:
\begin{equation}
  b_k=\mathrm{round}\!\left(
  B\frac{(v_k^{+})^{\gamma}|C_k|^{1-\gamma}}
  {\sum_j(v_j^{+})^{\gamma}|C_j|^{1-\gamma}}\right),\qquad \gamma=0.6.
  \label{eq:quota}
\end{equation}
Here $v_k$ is the mean conservative score of the top 10\% examples in cluster $C_k$, and $v_k^{+}=\max(v_k,0)$. For the 300K pool, $k=\max(50,\lfloor\sqrt{N}\rfloor)$ yields 547 clusters with about 548 examples per cluster on average; clustering is included in the 0.18 GPU-hours per PEFT--task application reported below. The final subset takes the top-$b_k$ examples by $q$ inside each cluster. Clusters with non-positive value receive no budget; deterministic remainder handling makes the quotas sum to $B$. For unseen configurations, a registry-defined structural policy distinguishes near-support, far-support, and unseen-family targets. Far targets use an optional residual calibration head when the native high-fidelity labeler supports their PEFT family.
