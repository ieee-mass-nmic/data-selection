A machine-learning platform often maintains several parameter-efficient fine-tuning (PEFT) configurations for different tasks and deployment budgets. Gradient-based data selectors preserve target specificity, but they rebuild a gradient datastore whenever the PEFT configuration changes. Reusing one subset avoids this computation but loses quality because different trainable subspaces value different examples. We present PCU-Select, which amortizes data selection across a PEFT registry. PCU-Select represents every supported PEFT family at 24 shared intervention sites: Transformer outputs through which PEFT updates affect the model. It caches reusable sample and task signals at these sites, then combines them with a code that describes the target PEFT. After a one-time offline stage, a single scorer ranks candidates for any supported PEFT--task pair without rebuilding gradients. On Llama-2-7B across four tasks and five seen PEFTs, PCU-Select achieves 35.18 average quality, nearly tying per-PEFT LESS at 35.14. Onboarding a new four-task PEFT costs 1.44 GPU-hours instead of 63.2, a 44-fold reduction. PCU-Select thus preserves the quality of per-PEFT selection while reducing its marginal cost at registry scale.
