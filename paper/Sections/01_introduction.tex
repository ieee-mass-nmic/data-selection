\section{Introduction}

Efficient adaptation of large language models requires reducing both the number of trainable parameters and the amount of training data. A machine-learning platform must also serve heterogeneous tasks and deployment constraints, and therefore rarely relies on a single adaptation setting. Instead, it keeps a \emph{registry} of PEFT configurations for different tasks, latency targets, and memory limits. The registry may include LoRA at several ranks and placements, adapters, (IA)$^3$, prefix tuning, and BitFit. Data-efficient fine-tuning complements these methods by selecting a compact subset from a larger instruction pool. In current practice, however, PEFT controls which parameters move while data selection independently chooses which examples drive the update.

Current data selectors rarely model this interaction. Quality filters, diversity heuristics, and proxy-model scores assign each example one value for a target task, then reuse the selected subset across adaptation methods. This PEFT-agnostic assumption ignores how each method changes the trainable subspace. LoRA injects low-rank additive updates into selected projections~\cite{hu2022lora}; adapters insert residual bottlenecks~\cite{houlsby2019adapters}; and (IA)$^3$ learns multiplicative activation scalings~\cite{liu2022ia3}. An example aligned with a task inside one subspace need not remain aligned inside another.

When one pool must serve a whole registry, this omission becomes a cost problem. LESS~\cite{xia2024less}, a strong gradient-based selector, builds a target-specific gradient datastore. It then ranks candidates by their influence on a \emph{task sketch}, a small validation set that represents the target task. Because the datastore depends on the target adapter's trainable parameters, each configuration must rebuild it. Selection cost therefore grows linearly with the number of PEFTs. The obvious shortcut selects once and reuses the subset everywhere. It removes the recomputation cost but also loses the target specificity that makes gradient selection valuable. Indeed, reusing a single LESS subset performs worse than a PEFT-agnostic representation baseline (Table~\ref{tab:quality-cost-headline}). Existing methods thus face a reuse-versus-recompute dilemma: they are either expensive for every target or effectively PEFT-agnostic.

Our motivation study shows that PEFT-specific utility changes with the trainable subspace. Figure~\ref{fig:motivation-disagreement} scores one candidate pool under LoRA, adapter, and (IA)$^3$ variants. Independent replicates under the same configuration agree more strongly than scoring runs across configurations. Same-family changes preserve partial structure, whereas cross-family changes produce the largest drop in rank agreement and Top-5\% overlap. The transfer matrix shows the downstream consequence. Selecting data for the training PEFT beats mismatched \emph{cross-family} selection by 2.0 points on average across target PEFTs (Appendix Table~\ref{tab:transfer}). The Analysis section reports a narrower 1.45-point gap within a LoRA configuration scan.

This paper treats data value as a PEFT-conditioned quantity. Given an example $x$, task $t$, and PEFT configuration $p$, we predict whether training on $x$ will help task $t$. This prediction is specific to the trainable subspace defined by $p$. PEFT-agnostic selection asks only whether $x$ is generally useful for the task. Per-PEFT influence selection captures the configuration but recomputes expensive supervision for every target. Our formulation retains the configuration condition while sharing supervision across targets.

PCU-Select (PEFT-Conditional Utility) separates reusable signals from target-specific decisions. It maps every supported PEFT family onto 24 \emph{intervention sites}: selected Transformer outputs through which PEFT updates affect the model. Samples and task sketches share this coordinate system, so PCU-Select can cache their signals once. A structured PEFT code then specifies where and how the target configuration acts. PCU-Select trains one conditional scorer from two label sources. Cheap gradient-proxy labels cover the candidate pool, while short-update labels correct consequential ranking errors. At deployment, the scorer combines the cached signals with the target code, ranks the pool, and spreads the budget across semantic clusters to avoid near-duplicates.

This design amortizes target-specific selection across a PEFT registry. Our contributions are three-fold:
\begin{itemize}
    \item \textbf{PEFT-conditioned utility.} We formulate data value as $u(x,p,t)$ and show that both data-value rankings and the downstream quality of selected subsets depend on the PEFT configuration (Figure~\ref{fig:motivation-disagreement}; Appendix Table~\ref{tab:transfer}).
    \item \textbf{Intervention-site amortization.} PCU-Select represents incompatible PEFT parameterizations through the same 24 module-output sites. It shares sample and task signals across configurations, while a structured PEFT code preserves target specificity without rebuilding a gradient datastore. Removing the PEFT code or task sketch reduces downstream performance by 1.37 or 0.82 points, respectively (Appendix Table~\ref{tab:ablations}).
    \item \textbf{Registry-scale quality and cost.} Across four tasks and five seen PEFTs on Llama-2-7B~\cite{touvron2023llama2}, PCU-Select gains 0.74 points over RDS+ and differs from per-PEFT LESS by $+0.04$. It reduces the selection-stage cost per added four-task PEFT by a factor of $44$ (1.44 versus 63.2 GPU-hours; Tables~\ref{tab:main-results} and~\ref{tab:quality-cost-headline}).
\end{itemize}
