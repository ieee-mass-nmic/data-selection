\section{Introduction}

Large language models (LLMs) acquire broad linguistic patterns and factual knowledge through self-supervised pre-training on massive text corpora. To make these general-purpose models follow user instructions, respect task-specific output formats, or specialize toward a target domain, downstream adaptation remains a central step in modern LLM development. The most direct adaptation strategy is full-parameter supervised fine-tuning, which updates every model parameter for each downstream task. However, as model sizes grow from billions to hundreds of billions of parameters, full fine-tuning becomes increasingly expensive: it requires substantial optimizer memory during training and produces a full model copy for every task, leading to high storage and deployment costs.

Parameter-efficient fine-tuning (PEFT) addresses this bottleneck by freezing most pre-trained weights and optimizing only a small set of task-specific parameters. Representative methods include adapters, which insert small bottleneck modules into Transformer blocks \cite{houlsby2019adapters}; LoRA, which injects trainable low-rank updates into linear projections \cite{hu2022lora}; and (IA)$^3$, which learns multiplicative vectors to rescale attention and feed-forward activations \cite{liu2022ia3}. These methods greatly reduce the number of trainable parameters and make LLM adaptation feasible under limited hardware budgets. Yet PEFT improves efficiency only on the model side. The success of fine-tuning also depends critically on the data side.

Recent studies show that the quantity of instruction data is not the sole determinant of fine-tuning performance. LIMA demonstrates that a 65B LLaMA model fine-tuned on only 1,000 carefully curated examples can exhibit strong instruction-following behavior without reinforcement learning from human feedback \cite{zhou2023lima}. Subsequent work further shows that filtering for high-quality, diverse, or task-relevant examples can match or even outperform training on much larger, noisier instruction corpora \cite{chen2024alpagasus, lu2023instag, liu2024deita, xia2024less}. This has motivated data-efficient fine-tuning, whose goal is to identify a small subset of valuable training examples from a large candidate pool.

Data efficiency and parameter efficiency are naturally complementary: the former reduces the number of examples and optimization steps, while the latter reduces the number of trainable parameters. However, existing data selection methods are usually designed as PEFT-agnostic procedures. They often assign one global data value score to each example based on quality, diversity, proxy-model feedback, or gradient similarity, and then reuse the selected subset across downstream adaptation settings. This assumption is convenient, but it overlooks a structural property of PEFT: different PEFT methods expose different trainable parameter subspaces. As a result, the same example may induce different effective update directions under LoRA, adapters, and (IA)$^3$. If data value is defined by its effect on the actual optimization trajectory, then data value should depend not only on the target task, but also on the target PEFT method.

\paragraph{Motivating question.}
This paper studies whether instruction data value is invariant across PEFT methods. We focus on three PEFT methods with distinct adaptation mechanisms: attention-only LoRA, bottleneck adapters, and (IA)$^3$. Table~\ref{tab:peft-configs} summarizes their update spaces. LoRA modifies attention projections through low-rank matrices; adapters add residual nonlinear transformations; and (IA)$^3$ only rescales internal activations. These design differences suggest that a training example that is useful for one PEFT method may not be equally useful for another.

To examine this question, we conduct a motivation study using Qwen2.5-1.5B-Base as the frozen backbone. We sample 20K instruction examples from Tulu-v2-sft-mixture as the candidate pool and use 128 GSM8K training examples as a target development set. For each PEFT method $p$, we compute per-example gradients only with respect to its trainable parameters $\theta_p$. Given a candidate example $x_i$ and a target development set $T$, we compute
\begin{equation}
    g_i^{(p)} = \nabla_{\theta_p} \ell(x_i; \theta_p), \qquad
    g_T^{(p)} = \frac{1}{|T|}\sum_{z \in T} \nabla_{\theta_p} \ell(z; \theta_p),
\end{equation}
where the backbone is frozen and the loss is computed only on assistant-response tokens. Following the spirit of gradient-based data selection methods such as LESS \cite{xia2024less}, we project gradients into a low-dimensional space with a random projection $R_p(\cdot)$ and score each example by
\begin{equation}
    s_i^{(p)} = \cos\left(R_p(g_i^{(p)}), R_p(g_T^{(p)})\right).
\end{equation}
This diagnostic score measures how well a candidate example aligns with the target task under the trainable subspace of PEFT method $p$.

Figure~\ref{fig:ranking-disagreement} evaluates whether the resulting data rankings transfer across PEFT methods. We compare the score sequences using Spearman rank correlation and compare the selected subsets using Top-5\% overlap. The off-diagonal correlations are much lower than the diagonal entries, and the Top-5\% overlaps remain limited. Since two random 5\% subsets have an expected overlap of 5\%, cross-PEFT overlaps above random still indicate some shared notion of generally useful data; however, their distance from 100\% and from same-PEFT split-half stability shows that the selected subsets are far from interchangeable. This suggests that data value is not PEFT-invariant.

Ranking disagreement alone does not necessarily imply downstream performance differences. Therefore, we further train each PEFT method on the Top-5\% subsets selected by every PEFT method, producing the cross-PEFT transfer matrix in Figure~\ref{fig:cross-peft-transfer}. All cells in the same row use the same training PEFT configuration and hyperparameters; only the selected data subset changes. The matrix exhibits diagonal dominance: LoRA performs best or near-best on LoRA-selected data, adapters on adapter-selected data, and (IA)$^3$ on (IA)$^3$-selected data. This pattern indicates that cross-PEFT data transfer is incomplete. In other words, data selected under one PEFT method is not always optimal for another PEFT method, even when the backbone, candidate pool, target task, and selection budget are fixed.

These observations motivate PEFT-aware data selection. Rather than treating data value as a universal property of an example, we argue that data value should be conditioned on the target adaptation mechanism. A PEFT-aware selector should estimate whether an example is useful under the trainable parameter subspace that will actually be optimized. This view unifies data-efficient and parameter-efficient fine-tuning: efficient LLM adaptation requires selecting not only a small set of parameters, but also a small set of examples that are compatible with those parameters.

\paragraph{Contributions.}
We make the following contributions:
\begin{itemize}
    \item We formulate the problem of PEFT-aware instruction data selection, where the utility of a candidate example is conditioned on both the target task and the target PEFT method.
    \item We provide a diagnostic motivation study showing that LoRA, adapters, and (IA)$^3$ induce different data-value rankings and select different high-value subsets from the same instruction pool.
    \item We show that ranking disagreement has practical consequences through a cross-PEFT transfer matrix, where each PEFT method benefits most from data selected under its own trainable subspace.
    \item We identify the gap between existing PEFT-agnostic data selection methods and the requirements of joint data- and parameter-efficient LLM adaptation, motivating PEFT-conditioned data-value modeling.
\end{itemize}
