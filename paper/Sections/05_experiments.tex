\section{Experiments}

\subsection{Experimental Setup}

We evaluate the proposed method on data-efficient fine-tuning tasks under multiple PEFT configurations. The main goal is to answer the following questions:
\begin{itemize}
    \item Does PEFT-conditioned data selection outperform PEFT-agnostic or random selection under the same data budget?
    \item Can the learned scorer transfer across different PEFT methods?
    \item How important are the PEFT representation and task representation in the scorer?
    \item Can the method generalize to unseen PEFT configurations?
\end{itemize}

\subsection{Datasets and Tasks}

We conduct experiments on instruction-following and reasoning-oriented datasets. The candidate training data is sampled from the original training set, and the selected subset is used for PEFT fine-tuning. The downstream performance is evaluated on held-out validation or test sets.

The main evaluation tasks include:
\begin{itemize}
    \item GSM8K for mathematical reasoning.
    \item Additional instruction-following or commonsense reasoning datasets.
    \item Optional cross-domain datasets for testing transferability.
\end{itemize}

\subsection{PEFT Configurations}

We consider multiple PEFT methods, including LoRA, Prefix Tuning, and IA3. Each method uses the same pretrained base model and comparable training budgets whenever possible. For each target PEFT method, data selection is performed before fine-tuning.

Example configurations include:
\begin{itemize}
    \item LoRA with rank $r$ and scaling factor $\alpha$.
    \item Prefix Tuning with a fixed number of virtual tokens.
    \item IA3 with learned activation scaling vectors.
\end{itemize}

\subsection{Baselines}

We compare our method with the following baselines:
\begin{itemize}
    \item Random: randomly selects samples under the same budget.
    \item Loss-based selection: selects samples according to training loss.
    \item Uncertainty-based selection: selects samples according to prediction uncertainty.
    \item IFD-based selection: selects samples according to instruction-following difficulty.
    \item PEFT-agnostic scorer: removes PEFT condition from our scorer.
\end{itemize}

\subsection{Implementation Details}

All methods use the same base model, tokenizer, training epochs, maximum sequence length, and evaluation protocol unless otherwise specified. For PEFT fine-tuning, we keep the pretrained model frozen and update only PEFT parameters. The data budget is controlled by the retention ratio, such as 10\%, 20\%, or 30\% of the full training set.

The utility scorer is trained on source PEFT configurations and then evaluated on target PEFT configurations. For unseen PEFT experiments, the target PEFT method is excluded from scorer training.
