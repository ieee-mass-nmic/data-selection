\section{Problem Definition and Design Objectives}
\label{sec:problem-definition}

Instruction fine-tuning for large language models typically relies on a large candidate data pool. However, different parameter-efficient fine-tuning (PEFT) configurations do not prefer the same training samples. For example, attention-layer LoRA, prefix tuning, IA3, and Adapters differ in their internal intervention sites, trainable capacity, and perturbation form. Therefore, selecting data only according to sample-level quality or perplexity on a single target task is insufficient for modeling the conditional utility among samples, PEFT configurations, and target tasks.

Let the candidate sample pool be
\begin{equation}
  \mathcal{D}=\{x_i\}_{i=1}^{N},
\end{equation}
where each sample $x_i$ consists of an instruction, a response, and a source tag. Given a target task $t^{*}$, a target PEFT configuration $p^{*}$, and a data budget $B$, the goal is to select a subset
\begin{equation}
  \mathcal{S}^{*}\subseteq \mathcal{D}, \qquad |\mathcal{S}^{*}|=B,
\end{equation}
such that fine-tuning with $\mathcal{S}^{*}$ under the target configuration $p^{*}$ yields a lower loss or a higher task metric on the validation or test set of task $t^{*}$. Exhaustively evaluating the true training gain of every candidate sample under every PEFT configuration is prohibitively expensive. Hence, a reusable, generalizable, and condition-aware data selector is required.

This chapter introduces PCU-Select. Its core idea is to first construct a low-cost low-fidelity utility $u^{\mathrm{lo}}$ and a small number of costly high-fidelity utilities $u^{\mathrm{hi}}$ in the offline stage, and use them to train a conditional Scorer. In the application stage, the Scorer takes the target PEFT representation $z_{p^{*}}$ and the target task representation $z_{t^{*}}$ as conditions, and outputs a predicted utility mean $\hat{\mu}(x,p^{*},t^{*})$ and uncertainty $\hat{\sigma}(x,p^{*},t^{*})$ for each candidate sample $x$. A cluster-quota mechanism is then used to select a subset that balances high utility and semantic coverage.

PCU-Select is designed around four objectives. First, \textbf{conditionality}: the scoring function explicitly takes the sample, PEFT configuration, and task sketch as inputs instead of learning a PEFT-agnostic universal sample ranker. Second, \textbf{reusability}: the offline sample features, gradient signatures, and Scorer can be reused across multiple target PEFT configurations or tasks. Third, \textbf{multi-fidelity learning}: low-fidelity signals provide large-scale coverage, while high-fidelity short-horizon updates correct the bias of the low-fidelity proxy. Fourth, \textbf{controlled cost}: the application stage only requires batched forward scoring, semantic clustering, and within-cluster ranking, which is substantially cheaper than repeating sample-level short-horizon fine-tuning for every target configuration.

\section{Overall Framework}
\label{sec:overall-framework}

As shown in Figure~\ref{fig:pcu-select-architecture}, PCU-Select consists of an offline meta-training stage and an online application stage.

\subsection{Offline Meta-Training Stage}
\label{subsec:offline-stage}

The offline stage takes as input a meta candidate sample pool $\mathcal{D}_{\mathrm{meta}}$, a PEFT configuration space $\mathcal{P}$, a task set $\mathcal{T}$, a selector model for feature and gradient-signature extraction, and a global configuration $\mathcal{C}$. The offline stage proceeds as follows:

\begin{enumerate}
  \item Extract a semantic representation, difficulty statistics, and a layerwise activation signature for each sample $x$, obtaining the sample representation $z_x$.
  \item Compute the sample gradient signature $g_x^{\omega}$ on a predefined set of intervention sites $\Omega$, and reduce its dimension using fixed random projection matrices.
  \item Encode the intervention sites, capacity, training recipe, and optional functional fingerprint of each PEFT configuration $p$, obtaining the PEFT representation $z_p$.
  \item Construct a task sketch $V_t$ for each task $t$ using only its training or development split, and encode it into a task representation $z_t$ and a task gradient signature $g_t^{\omega}$.
  \item Compute the low-fidelity utility $u^{\mathrm{lo}}(x,p,t)$ from the PEFT-conditional weights $\tilde{\alpha}_p^{\omega}$ and gradient cosine similarities.
  \item Sample a small number of triples $(x,p,t)$ and compute their high-fidelity utility $u^{\mathrm{hi}}(x,p,t)$ using PEFT-only short-horizon updates.
  \item Pretrain the Scorer with low-fidelity labels, and then jointly fine-tune it with high-fidelity labels to obtain a reusable conditional utility predictor.
\end{enumerate}

\subsection{Online Application Stage}
\label{subsec:apply-stage}

The online application stage targets a specific pair $(p^{*},t^{*})$. The representation $z_{p^{*}}$ is first computed from the target PEFT configuration, and $z_{t^{*}}$ is computed from the target task sketch. The Scorer then outputs the utility mean and uncertainty for each sample in the candidate pool, from which a conservative score is constructed:
\begin{equation}
  q(x) = \hat{\mu}(x,p^{*},t^{*}) - \lambda \hat{\sigma}(x,p^{*},t^{*}),
  \label{eq:conservative-score}
\end{equation}
where $\lambda$ is the uncertainty penalty coefficient, set to $0.2$ by default. Finally, candidate samples are clustered with MiniBatch K-Means in the semantic space $e_x^{\mathrm{joint}}$. The budget is allocated according to cluster-level utility, and the top-scoring samples are selected within each cluster.

\section{Intervention Sites and PEFT-Conditional Weights}
\label{sec:site-and-alpha}

\subsection{Default Intervention Site Set}
\label{subsec:intervention-sites}

PCU-Select does not compute sample gradients on all parameter tensors. Instead, it defines a stable set of intervention sites in the forward graph. For an $L$-layer decoder-only Transformer, PCU-Select first selects $k=8$ layer indices
\begin{equation}
  L_{\mathrm{sig}}=\operatorname{uniform\_indices}(L,k=8),
\end{equation}
for example $[3,7,11,15,19,23,27,31]$ when $L=32$. Three hook points are defined in each selected layer: the self-attention output $\mathrm{attn\_out}$, the MLP output $\mathrm{mlp\_out}$, and the block output after residual merging $\mathrm{block\_residual}$. The default site set is
\begin{equation}
  \Omega_{\mathrm{default}}=\{(l,m)\mid l\in L_{\mathrm{sig}},\;m\in\{\mathrm{attn\_out},\mathrm{mlp\_out},\mathrm{block\_residual}\}\},
  \label{eq:default-sites}
\end{equation}
with $|\Omega_{\mathrm{default}}|=24$.

Using module outputs rather than finer-grained positions such as $q\_\mathrm{proj}$, $k\_\mathrm{proj}$, or $v\_\mathrm{proj}$ is motivated by three considerations. First, different backbones may use fused QKV or split QKV implementations, making fine-grained hooks difficult to reuse consistently. Second, the self-attention output indirectly covers the combined training effects of query, key, value, and output projections. Third, 24 sites provide a practical trade-off between cache size and discriminative power.

\subsection{Mapping PEFT Configurations to Sites}
\label{subsec:peft-site-mapping}

Different PEFT algorithms act on the model in different ways. PCU-Select maps a PEFT configuration $p$ into a site mask $\operatorname{mask}(p,\omega)$ and an operator type $\operatorname{op}(p,\omega)$. Table~\ref{tab:peft-site-map} summarizes typical mapping rules.

\subsection{PEFT-Conditional Weights}
\label{subsec:alpha-weight}

The low-fidelity utility must account for how strongly the target PEFT acts on different sites. Let $\operatorname{mask}(p,\omega)\in\{0,1\}$ denote whether configuration $p$ acts on site $\omega$, $\operatorname{cap}(p,\omega)$ denote the trainable capacity associated with that site, $\operatorname{op}(p,\omega)$ denote the operator type, and $d_{\mathrm{model}}$ denote the backbone hidden dimension. The unnormalized PEFT-conditional weight is defined as
\begin{equation}
  \alpha_p^{\omega}
  =\operatorname{mask}(p,\omega)\cdot
  \rho_{\operatorname{op}(p,\omega)}\cdot
  \tanh\left(\eta\log\left(1+\frac{\operatorname{cap}(p,\omega)}{d_{\mathrm{model}}}\right)\right),
  \label{eq:alpha-raw}
\end{equation}
where $\eta$ controls capacity scaling and $\rho_{\operatorname{op}}$ is an operator prior weight. The default value is $\eta=1.0$, with
\begin{equation}
  \rho_{\mathrm{lowrank}}=1.0,\quad
  \rho_{\mathrm{multiplicative}}=0.6,\quad
  \rho_{\mathrm{bottleneck}}=0.8,\quad
  \rho_{\mathrm{prefix}}=0.4,\quad
  \rho_{\mathrm{bias}}=0.3.
\end{equation}

To prevent the low-fidelity utility from being dominated by the number of active sites, all site weights are normalized:
\begin{equation}
  \tilde{\alpha}_p^{\omega}
  =\frac{\alpha_p^{\omega}}{\max\left(\varepsilon,\sum_{\omega'\in\Omega}\alpha_p^{\omega'}\right)},
  \label{eq:alpha-normalized}
\end{equation}
where $\varepsilon$ is a small constant that prevents division by zero. The normalized $\tilde{\alpha}_p^{\omega}$ can be interpreted as the conditional distribution of configuration $p$ over the site set.

\section{Sample, PEFT, and Task Representations}
\label{sec:three-representations}

\subsection{Sample Representation}
\label{subsec:sample-representation}

The sample representation $z_x$ is obtained by concatenating a semantic representation, difficulty statistics, and a layerwise activation signature:
\begin{equation}
  z_x=[e_x^{\mathrm{joint}};d_x;a_x].
  \label{eq:sample-representation}
\end{equation}

The semantic representation $e_x^{\mathrm{joint}}$ is produced by feeding the concatenated instruction and response into a sentence-embedding model, with a default dimension of 768. This channel captures sample topic, task type, and response style, and also serves as the default space for semantic clustering in the application stage.

The difficulty statistics $d_x$ have a fixed dimension of 16. They include instruction length, response length, total length, response-to-instruction length ratio, the mean and variance of sample-level loss, maximum loss, perplexity, average log probability, entropy statistics, indicators for chain-of-thought, code, and question-answering samples, and a coarse language-type indicator. Loss, perplexity, log probability, and entropy are obtained from a single forward pass of the selector model.

The layerwise activation signature $a_x$ records eight statistics for each layer in $L_{\mathrm{sig}}$: the norms of $\mathrm{attn\_out}$, $\mathrm{mlp\_out}$, and $\mathrm{residual}$, attention entropy, attention-head norm variance, MLP activation norm, hidden-state token variance, and the dot product between the last and first token representations. The default dimension is $8\times 8=64$.

\subsection{PEFT Representation}
\label{subsec:peft-representation}

The PEFT representation consists of a site mask, a capacity vector, a training recipe vector, and an optional functional fingerprint:
\begin{equation}
  z_p=[m_p;c_p;r_p;f_p].
  \label{eq:peft-representation}
\end{equation}
Here $m_p\in\mathbb{R}^{|\Omega|\times 4}$, where the four elements in the second dimension indicate whether a site is active, whether it uses an additive operator, whether it uses a multiplicative operator, and whether it uses a prefix operator. Bottleneck Adapters and additive low-rank methods share the additive dimension in this encoding. Bias shift is identified by the pattern in which the site is active while all three operator bits are zero.

The capacity vector $c_p$ has a default dimension of 16. It contains the number of trainable parameters, the trainable-parameter ratio, normalized LoRA rank and alpha values, Adapter bottleneck size, prefix length, extra FLOPs, extra memory, inference-latency ratio, whether the KV cache is affected, the capacity share of each operator, and operator-count statistics. The training recipe vector $r_p$ also has a default dimension of 16 and encodes the learning rate, warmup ratio, weight decay, scheduler, optimizer, batch size, dropout, maximum steps, gradient clipping, and initialization method.

The functional fingerprint $f_p$ is optional. It is obtained by applying a small PEFT warmup to the base model on a task-agnostic probe set, and recording site-wise activation changes and downstream loss changes. This channel can improve discrimination for unseen PEFT configurations, but it introduces additional cost. Therefore, it is treated as an ablation factor in this thesis.

\subsection{Task Representation}
\label{subsec:task-representation}

The task condition is not constructed from the test set. Instead, a task sketch $V_t$ is sampled from the training or development split of the target task. The default sketch size is $N_V=32$. Samples are divided into short, medium, and long strata according to length quantiles, and sampled proportionally with a fixed random seed to avoid length-distribution bias. Each sketch sample $v\in V_t$ is encoded in the same way as candidate samples, yielding $z_v$. A Set Transformer-style attention pooling module is then used to produce the task representation $z_t$:
\begin{equation}
  H_t=\operatorname{MHA}(Q,K=\{z_v\}_{v\in V_t},V=\{z_v\}_{v\in V_t}),
\end{equation}
\begin{equation}
  z_t=\operatorname{MeanPool}(H_t),
  \label{eq:task-representation}
\end{equation}
where $Q\in\mathbb{R}^{q\times d}$ is a learnable latent query matrix, with $q=4,d=256$ by default. Compared with simple mean pooling, attention pooling reduces the influence of outlier sketch samples on the task representation.

\section{Site Gradient Signatures and Low-Fidelity Utility}
\label{sec:gradient-signature-lo}

\subsection{Sample Loss}
\label{subsec:sample-loss}

For instruction tuning, this thesis uses the autoregressive cross-entropy over response tokens as the default sample loss, masking out prompt or instruction tokens:
\begin{equation}
  \ell(x)=-\sum_{i\in\mathcal{M}_{\mathrm{resp}}}\log p_{\theta}(x_i\mid x_{<i}),
  \label{eq:sample-loss}
\end{equation}
where $\mathcal{M}_{\mathrm{resp}}$ denotes the set of response-token positions. For continued pretraining, the mask can be extended to all tokens. For preference learning, the interface reserves a DPO-style loss, although the main experiments focus on response language modeling.

\subsection{Sample Gradient Signature}
\label{subsec:sample-gradient-signature}

For each sample $x$, forward hooks are registered on the site set $\Omega$, and gradients are retained for the site activations $h^{\omega}$. After a forward pass computes the sample loss $\ell(x)$, backpropagation yields the raw gradient $\nabla_{h^{\omega}}\ell(x)$ at each site. The gradient is then pooled along the time dimension; by default, the mean is taken only over response tokens:
\begin{equation}
  \bar{g}_x^{\omega}=\frac{1}{|\mathcal{M}_{\mathrm{resp}}|}\sum_{i\in\mathcal{M}_{\mathrm{resp}}}\nabla_{h_i^{\omega}}\ell(x).
  \label{eq:gradient-pooling}
\end{equation}

To reduce cache size and make dimensions consistent across sites, a site-specific random projection matrix $\Phi_{\omega}\in\mathbb{R}^{d_{\mathrm{proj}}\times d_{\mathrm{model}}}$ projects the pooled gradient vector to $d_{\mathrm{proj}}=256$ dimensions:
\begin{equation}
  g_x^{\omega}=\frac{\Phi_{\omega}\bar{g}_x^{\omega}}{\left\lVert\Phi_{\omega}\bar{g}_x^{\omega}\right\rVert_2}.
  \label{eq:sample-gradient-signature}
\end{equation}
The random projection matrix is determined by $\operatorname{hash}(\omega,\mathrm{global\_seed})$ and is persisted after generation, ensuring signature consistency across runs.

\subsection{Task Gradient Signature}
\label{subsec:task-gradient-signature}

The gradient signature of a task sketch $V_t$ is obtained by mean-pooling the signatures of its sketch samples:
\begin{equation}
  g_t^{\omega}=\operatorname{Normalize}\left(\frac{1}{|V_t|}\sum_{v\in V_t}g_v^{\omega}\right).
  \label{eq:task-gradient-signature}
\end{equation}
Sample signatures and task signatures must be computed using the same selector model, so that site coordinates and gradient scales remain consistent. The selector model is chosen as the smallest available model in the same backbone family, providing a low-cost approximation of the relative training directions of samples across intervention sites.

\subsection{Low-Fidelity Utility}
\label{subsec:lo-utility}

Given a sample $x$, a PEFT configuration $p$, and a task $t$, the low-fidelity utility is defined as the PEFT-conditionally weighted site gradient similarity:
\begin{equation}
  u^{\mathrm{lo}}(x,p,t)=\sum_{\omega\in\Omega}\tilde{\alpha}_p^{\omega}\cdot
  \cos\left(g_x^{\omega},g_t^{\omega}\right).
  \label{eq:lo-utility}
\end{equation}
This quantity measures whether the gradient direction of sample $x$ aligns with the task sketch on the sites emphasized by configuration $p$. If a sample's gradient is highly aligned with the task-sketch gradient on the main sites affected by the target PEFT, the sample is more likely to reduce the target task loss during short-horizon fine-tuning. Since $u^{\mathrm{lo}}$ only requires one sample-level backward pass and cached signatures can be reused, it is suitable for constructing pretraining signals over a large number of triples.

\section{High-Fidelity Utility and Multi-Fidelity Label Construction}
\label{sec:hi-utility}

\subsection{Anchor Checkpoints}
\label{subsec:anchor-checkpoint}

The low-fidelity utility cannot fully simulate the task gain after an actual PEFT update, so PCU-Select uses a small number of high-fidelity labels to calibrate the Scorer. High-fidelity computation is based on two anchor checkpoints: the base model $\theta_{\mathrm{base}}$ and a warm checkpoint $\theta_{\mathrm{warm}}$. The warm checkpoint is obtained by randomly sampling 1000 examples from the meta sample pool and training a base-configuration LoRA on attention layers for 200 steps. Both anchors are frozen, and short-horizon updates optimize only PEFT parameters.

\subsection{PEFT-Only Short-Horizon Update}
\label{subsec:short-update}

For a triple $(x,p,t)$, an anchor $a$, and a horizon $h$, PCU-Select starts from freshly initialized PEFT parameters specified by configuration $p$, and performs $h$ update steps on the single sample $x$ using the optimizer and recipe specified by $p$. Let $\operatorname{Adapt}_{p}^{h}(x;\theta_a)$ denote the model after applying this short-horizon PEFT update on anchor $\theta_a$. The high-fidelity gain is defined as the decrease in task-sketch loss:
\begin{equation}
  \Delta(x,p,t,a,h)=L_{V_t}(\theta_a)-L_{V_t}\left(\theta_a+\operatorname{Adapt}_{p}^{h}(x;\theta_a)\right),
  \label{eq:hi-delta}
\end{equation}
where
\begin{equation}
  L_{V_t}(\theta)=\frac{1}{|V_t|}\sum_{v\in V_t}\ell_{\theta}(v).
\end{equation}

To reduce noise, RankNorm is applied to $\Delta$ within the same $(p,t)$ bucket, converting it into a relative utility in $[0,1]$. For different horizons, this thesis uses $h=1$ and $h=4$ by default, and combines them as
\begin{equation}
  u^{\mathrm{hi}}=0.4\cdot \operatorname{RankNorm}(\Delta_{h=1})+0.6\cdot \operatorname{RankNorm}(\Delta_{h=4}).
  \label{eq:hi-utility}
\end{equation}

\subsection{High-Fidelity Triple Sampling}
\label{subsec:hi-sampling}

The high-fidelity label budget is denoted by $Q_H$, with $Q_H=10\mathrm{k}$ by default. To balance coverage and error correction under a limited budget, PCU-Select uses a three-stage sampling strategy:
\begin{enumerate}
  \item \textbf{Coverage sampling}: $50\%$ of the budget is used for stratified sampling across sample clusters, PEFT families, and PEFT capacity buckets, producing the initial high-fidelity training set.
  \item \textbf{Uncertainty sampling}: $30\%$ of the budget is allocated according to the current Scorer uncertainty and the low-fidelity utility, with query score
  \begin{equation}
    q_{\mathrm{query}}=\hat{\sigma}\cdot\left(1+\gamma\operatorname{ReLU}(\hat{u}^{\mathrm{lo}})\right),\qquad \gamma=0.5.
  \end{equation}
  \item \textbf{Boundary sampling}: $20\%$ of the budget focuses on triples where the Scorer ranking disagrees with the true ranking in the top-$k$ region, correcting the final selection boundary.
\end{enumerate}
The second and third stages rely on an initial Scorer trained from the first-stage labels.

\section{Conditional Scorer Model}
\label{sec:scorer}

\subsection{Model Architecture}
\label{subsec:scorer-architecture}

The Scorer takes a sample representation $z_x$, a PEFT representation $z_p$, and a task representation $z_t$, and outputs a utility mean $\hat{\mu}$ and uncertainty $\hat{\sigma}$:
\begin{equation}
  (\hat{\mu},\hat{\sigma})=F_{\psi}(z_x,z_p,z_t).
\end{equation}

The three inputs are first encoded by separate multilayer perceptrons:
\begin{align}
  h_x &= f_x(z_x)\in\mathbb{R}^{256},\\
  h_p &= f_p(z_p)\in\mathbb{R}^{128},\\
  h_t &= f_t(z_t)\in\mathbb{R}^{256}.
\end{align}
Here $f_x$ and $f_t$ consist of LayerNorm, a linear layer, GELU, another linear layer, and LayerNorm. The PEFT branch $f_p$ uses a smaller hidden dimension to match the scale of the PEFT representation.

Conditional fusion is then applied to the sample representation. Let
\begin{equation}
  c=[h_p;h_t]\in\mathbb{R}^{384}.
\end{equation}
A linear layer generates FiLM modulation parameters $\gamma(c)$ and $\beta(c)$:
\begin{equation}
  h_{\mathrm{film}}=\gamma(c)\odot h_x+\beta(c).
  \label{eq:film-fusion}
\end{equation}
Meanwhile, a bilinear term explicitly models the interaction between the sample and the PEFT configuration:
\begin{equation}
  h_{\mathrm{bilin}}=\operatorname{Bilinear}(h_x,h_p)\in\mathbb{R}^{64}.
\end{equation}
The concatenation of $h_{\mathrm{film}}$ and $h_{\mathrm{bilin}}$ is then passed to the output head:
\begin{equation}
  h_o=\operatorname{MLP}([h_{\mathrm{film}};h_{\mathrm{bilin}}]),
\end{equation}
\begin{equation}
  \hat{\mu}=W_{\mu}h_o+b_{\mu},\qquad
  \hat{\sigma}=\operatorname{softplus}(W_{\sigma}h_o+b_{\sigma})+10^{-3}.
  \label{eq:mu-sigma-head}
\end{equation}
The model contains approximately 1.5M parameters and is suitable for batched forward scoring in the application stage.

\subsection{Training Objective}
\label{subsec:scorer-loss}

The Scorer training loss consists of a ranking loss, a regression loss, a low-fidelity proxy loss, and a heteroscedastic uncertainty loss:
\begin{equation}
  \mathcal{L}=\lambda_{\mathrm{rank}}\mathcal{L}_{\mathrm{rank}}^{\mathrm{hi}}
  +\lambda_{\mathrm{reg}}\mathcal{L}_{\mathrm{reg}}^{\mathrm{hi}}
  +\lambda_{\mathrm{proxy}}\mathcal{L}_{\mathrm{proxy}}^{\mathrm{lo}}
  +\lambda_{\mathrm{unc}}\mathcal{L}_{\mathrm{unc}}^{\mathrm{hi}}.
  \label{eq:scorer-total-loss}
\end{equation}
The default weights are $\lambda_{\mathrm{rank}}=1.0$, $\lambda_{\mathrm{reg}}=0.3$, $\lambda_{\mathrm{proxy}}=0.5$, and $\lambda_{\mathrm{unc}}=0.2$.

The ranking loss constructs sample pairs $(i,j)$ within the same $(p,t)$ bucket:
\begin{equation}
  \mathcal{L}_{\mathrm{rank}}
  =-\log\operatorname{sigmoid}\left((\hat{\mu}_i-\hat{\mu}_j)\cdot
  \operatorname{sgn}(u_i^{\mathrm{hi}}-u_j^{\mathrm{hi}})\right).
  \label{eq:rank-loss}
\end{equation}
The regression loss uses mean squared error or Huber loss to fit the high-fidelity utility. The low-fidelity proxy loss constrains the Scorer to learn the coarse structure of $z_x\times z_p\times z_t$ from a large number of low-fidelity samples. The heteroscedastic uncertainty loss is
\begin{equation}
  \mathcal{L}_{\mathrm{unc}}
  =\frac{1}{2}\frac{(u^{\mathrm{hi}}-\hat{\mu})^2}{\hat{\sigma}^2}+\frac{1}{2}\log\hat{\sigma}^2.
  \label{eq:unc-loss}
\end{equation}

\subsection{Two-Stage Training}
\label{subsec:scorer-training}

Training proceeds in two stages. Stage A uses only $u^{\mathrm{lo}}$ labels for low-fidelity pretraining, for 3 epochs with AdamW and learning rate $3\times10^{-4}$. Stage B adds high-fidelity labels and enables all loss terms, for 2 epochs with a reduced learning rate of $1\times10^{-4}$. This strategy prevents sparse and noisier high-fidelity labels from dominating the early stage of training, allowing the Scorer to first learn a stable conditional representation structure and then use high-fidelity signals to correct ranking boundaries.

\section{Data Selection Algorithm in the Application Stage}
\label{sec:apply-selection}

\subsection{Uncertainty-Aware Scoring}
\label{subsec:uncertainty-aware-score}

For a target configuration $(p^{*},t^{*})$, the Scorer outputs $\hat{\mu}_i$ and $\hat{\sigma}_i$ for each candidate sample. The conservative score follows Eq.~\eqref{eq:conservative-score}, where the uncertainty penalty reduces the probability that OOD or high-variance samples enter the final subset.

\subsection{Semantic Clustering and Adaptive Quotas}
\label{subsec:adaptive-quota}

Globally ranking only by $q(x)$ can select many semantically similar samples and lead to insufficient subset coverage. Therefore, PCU-Select applies MiniBatch K-Means clustering in the semantic space $e_x^{\mathrm{joint}}$. The number of clusters is
\begin{equation}
  k=\max(50,\lfloor\sqrt{N}\rfloor).
  \label{eq:kmeans-k}
\end{equation}
For each cluster $C_k$, cluster-level utility is defined as the mean score of the top $10\%$ samples in that cluster:
\begin{equation}
  v_k=\operatorname{Mean}\left(\operatorname{Top}_{10\%}\{q_i:x_i\in C_k\}\right).
  \label{eq:cluster-utility}
\end{equation}
Given the total budget $B$, the quota of cluster $C_k$ is
\begin{equation}
  b_k=\operatorname{round}\left(
  B\cdot\frac{(v_k^{+})^{\alpha}|C_k|^{1-\alpha}}{Z}
  \right),\qquad \alpha=0.6,
  \label{eq:cluster-quota}
\end{equation}
where $v_k^{+}=\max(v_k,0)$ and $Z$ is a normalization factor. The parameter $\alpha$ controls the trade-off between utility and cluster size: $\alpha=0$ reduces to allocation by cluster size, while $\alpha=1$ reduces to allocation by cluster utility. If $b_k>|C_k|$, it is clipped to $|C_k|$, and the remaining budget is redistributed to other clusters in proportion to $v_k^{+}$. The final subset selects the top-$b_k$ samples by $q_i$ within each cluster.

\subsection{OOD Detection and Calibration}
\label{subsec:ood-calibration}

The Scorer is trained primarily on the offline PEFT space $\mathcal{P}$. For a new target PEFT $p^{*}$, PCU-Select must determine whether it lies within the training-configuration distribution. After training, the mean $\mu_{z_p}$ and covariance $\Sigma_{z_p}$ of the training PEFT representations are recorded, and the Mahalanobis distance of the target configuration is computed:
\begin{equation}
  d^2(p^{*})=(z_{p^{*}}-\mu_{z_p})^{\top}\Sigma_{z_p}^{-1}(z_{p^{*}}-\mu_{z_p}).
  \label{eq:mahalanobis}
\end{equation}
The threshold $\tau_{\mathrm{id}}$ is set to the $95\%$ quantile of the distances of training PEFT configurations. If $d^2(p^{*})>\tau_{\mathrm{id}}$, calibration mode is triggered: 500 candidate samples are drawn, lightweight high-fidelity labels are computed for the target configuration, the Scorer body is frozen, and only a linear calibration head is fitted:
\begin{equation}
  \mu_{\mathrm{cal}}=\hat{\mu}+W_{\mathrm{cal}}[z_x;z_{p^{*}};z_t]+b_{\mathrm{cal}}.
  \label{eq:calibration-head}
\end{equation}
Full-pool scoring then uses $\mu_{\mathrm{cal}}$ instead of $\hat{\mu}$.

\section{Caching, Complexity, and Reproducibility}
\label{sec:cache-complexity}

PCU-Select decomposes expensive computation into cacheable features, signatures, and labels. Sample features are stored as parquet or arrow files. Site gradient signatures are sharded by site and stored as array files. Task sketches, task gradient signatures, PEFT configurations, low-fidelity labels, high-fidelity labels, and Scorer checkpoints are managed using content hashes. The core cache keys include
\begin{align}
  \mathrm{feature\_key} &= \operatorname{sha256}(\mathrm{selector\_model\_id},\mathrm{sample\_id},\mathrm{feat\_version}),\\
  \mathrm{grad\_key} &= \operatorname{sha256}(\mathrm{selector\_model\_id},\mathrm{sample\_id},\omega,\mathrm{global\_seed}),\\
  \mathrm{hi\_key} &= \operatorname{sha256}(a,p,x,t,h,\mathrm{seed}).
\end{align}

The main offline costs come from sample feature extraction, site gradient signatures, and high-fidelity short-horizon updates. The main online costs are batched Scorer forward passes, clustering, and ranking. Since low-fidelity signatures and the Scorer can be reused across multiple target PEFT configurations and tasks, the offline cost can be amortized as the number of target configurations increases. Chapter~\ref{chap:experiments} further quantifies this amortization benefit through break-even analysis.

\section{Chapter Summary}
\label{sec:method-summary}

This chapter presented PCU-Select, a PEFT-conditional data selection method. By using intervention sites as a bridge, PCU-Select aligns sample gradients, PEFT intervention locations, and task sketches in a comparable representation space. It constructs multi-fidelity training signals from a low-fidelity gradient proxy and high-fidelity short-horizon updates, builds a conditional Scorer with FiLM modulation and bilinear interactions, and finally selects data subsets for the target PEFT and task through uncertainty-aware scoring and semantic cluster quotas. The next chapter designs experiments to evaluate effectiveness, the necessity of conditioning, cost benefits, and generalization.
