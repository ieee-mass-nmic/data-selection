# Complete PCU-Select Design Plan (Implementation Version)

> This document is an implementation refinement of [pcu_select_research_plan.md](pcu_select_research_plan.md). It strengthens details that were not fully specified or not directly implementable in the original plan, and makes convergent revisions to several design points. The research motivation, overall strategy, and experimental framework follow the original plan and are not repeated here; this document only specifies the interfaces and algorithms at the executable layer.

This ensures that caches do not change with scheduling order and that resumable computation is valid.

## 2. Precise Definition of Intervention Sites Ω

### 2.1 Default Site Configuration

For a decoder-only transformer with L layers, define:

* Select the layer index set `L_sig = uniform_indices(L, k=8)`, e.g., for L=32: `[3,7,11,15,19,23,27,31]`.
* Use 3 hook points per layer:

  * `attn_out`: output of the self-attention module (before residual addition)
  * `mlp_out`: output of the MLP module (before residual addition)
  * `block_residual`: output after residual merging in this block (before layer norm)

This gives the default site set:

```
Ω_default = { (l, m) : l ∈ L_sig, m ∈ {attn_out, mlp_out, block_residual} }
|Ω_default| = 24
```

### 2.2 Why Not Use Finer Locations Such as `q_proj/k_proj/v_proj`?

* Fused QKV vs. split QKV differs across backbones, making hook reuse cumbersome;
* self-attention OUT consistently covers the training effects of q/k/v/o;
* 24 sites already strike a reasonable balance between cache size (≈2.5GB for N=100k) and discriminative power.

### 2.3 Mapping from PEFT to Sites

| PEFT                      | Affected site type                             | operator type       |
| ------------------------- | ---------------------------------------------- | ------------------- |
| LoRA on q_proj/v_proj     | `attn_out` (indirectly)                        | additive_low_rank   |
| LoRA on up_proj/down_proj | `mlp_out`                                      | additive_low_rank   |
| IA3 on attention          | `attn_out`                                     | multiplicative      |
| IA3 on FFN                | `mlp_out`                                      | multiplicative      |
| Bottleneck Adapter        | `block_residual`                               | additive_bottleneck |
| Prefix tuning             | all `attn_out` sites (prefix attention effect) | prefix              |
| BitFit                    | all sites                                      | bias_shift          |

The concrete implementation of `site_mask_of(PEFTConfig) -> dict[SiteID, float]` is in [`src/pcu_select/peft_space/site_mask.py`](../src/pcu_select/peft_space/site_mask.py).

## 3. Executable Formula for PEFT Conditional Weight `α_p^ω`

The original plan described it as "determined by PEFT site mask, capacity, and operator type"; this document gives the precise form.

Let:

* `mask(p, ω) ∈ {0, 1}`: whether PEFT p acts on site ω;
* `cap(p, ω)`: number of trainable parameters at that site, e.g., for LoRA at the layer, `2 * rank * d_model`;
* `op(p, ω) ∈ {additive_low_rank, multiplicative, additive_bottleneck, prefix, bias_shift}`;
* `η`, `ρ_op`: hyperparameters controlling capacity scaling and operator-type weights.

Then:

```
α_p^ω = mask(p, ω) · ρ_{op(p,ω)} · tanh( η · log(1 + cap(p, ω) / d_model) )
```

Normalize over the full site set:

```
α̃_p^ω = α_p^ω / max(ε, Σ_{ω'} α_p^{ω'})
```

This prevents the final low-fidelity utility `u^lo` from being dominated by the number of sites, and instead reflects the "distributional weight" over sites.

Default hyperparameters:

* `η = 1.0`
* `ρ = {additive_low_rank: 1.0, multiplicative: 0.6, additive_bottleneck: 0.8, prefix: 0.4, bias_shift: 0.3}`

Meaning of `ρ_op`: starting from equal capacity at the same site, different operators do not produce equally effective perturbations to training dynamics. These values serve as prior initial values and can be tuned in experiments or absorbed into scorer learning.

## 4. Definition of Sample Loss ℓ(x)

Determined by `TaskConfig.loss_type`:

* **`response_lm`** (instruction tuning, default): cross-entropy over response tokens, i.e., tokens after the prompt; instruction tokens are masked out.
* **`full_lm`** (continued pretraining style): cross-entropy over all tokens.
* **`preference`**: DPO-style preference-pair sample loss. Not the focus of this project, but the interface is reserved.

```
ℓ(x) = - Σ_{i ∈ response_mask} log p_θ(x_i | x_{<i})
```

Task sketch loss `L_V`: sum of ℓ(v) over sketch samples, averaged by the number of samples.

## 5. Site Gradient Signatures

### 5.1 Computation Procedure

```
for each sample x:
  with hooks at each ω ∈ Ω registering retain_grad on h^ω:
    forward(x), compute ℓ(x)
    ℓ.backward()
    for each ω: g_raw_ω = h^ω.grad   # shape: (B, T_x, d_model)
                g_pool_ω = pool_over_T(g_raw_ω)  # (B, d_model)
  for each ω:
    g_x^ω = Φ_ω @ g_pool_ω           # (B, d_proj)
    g_x^ω = g_x^ω / ||g_x^ω||₂
```

### 5.2 Temporal Pooling

* Default: `mean_over_response_tokens` (aligned with the loss mask).
* Alternatives: `last_token` or `attention_weighted` (for ablation).

### 5.3 Random Projection Matrices

* `Φ_ω ∈ R^{d_proj × d_model}`, one per site;
* `seed_ω = hash(site_id, global_seed)`;
* Entries sampled from `N(0, 1/d_proj)` (Johnson-Lindenstrauss scaling);
* Generated once and persisted permanently to avoid inconsistency across repeated runs.

### 5.4 Task Sketch Signatures

For sketch V, first compute `g_v^ω` for each sample, then pool:

```
g_t^ω = normalize( (1/|V|) · Σ_{v ∈ V} g_v^ω )
```

> Note: the model used for sketch signature computation is the **same selector model** as the one used for sample signatures (see §6.1), not an arbitrary backbone.

### 5.5 Low-Fidelity Utility

```
u^lo(x, p, t) = Σ_ω α̃_p^ω · cos(g_x^ω, g_t^ω)
```

## 6. Choice of Selector Model

### 6.1 Which Model Is Used to Extract Gradient Signatures?

**Key decision**: do site gradient signatures need to share the same source as the target backbone?

This plan chooses **selector model = the smallest available model in the backbone family** (e.g., LLaMA 7B rather than 65B), for the following reasons:

1. Gradient signatures are relative quantities; they mainly capture "which sites care about this sample" and do not require absolute precision;
2. Smaller models save time and memory;
3. Same family ensures consistent layer-index semantics;
4. PEFT fine-tuning at the large-model stage is a separate matter and does not depend on the selector model.

If experiments show insufficient performance, it can be upgraded to the same size as the target backbone; this is an ablation option.

### 6.2 Handling Differences from the Target Backbone

* `L_sig` is the layer index set on the selector model; different backbones do not directly share site coordinates.
* Therefore, one offline training run targets only one backbone family (e.g., LLaMA-7B/13B); cross-family transfer is not within the main claim.

## 7. Precise Construction of Sample Feature z_x

### 7.1 Semantic Representation `e_x`

* `e_x.instr`: encode the instruction using `sentence-transformers/all-mpnet-base-v2` or an equivalent model (768 dimensions).
* `e_x.resp`: encode the response in the same way.
* `e_x.joint`: concatenate instruction + response and encode them together (truncated to 512 tokens), used as the default channel for clustering and the task pool.
* `e_x.source_id`: optional `OneHot(source_dataset)`.

The main input to the scorer is `e_x.joint`. Other fields are retained for ablations.

### 7.2 Difficulty Statistics `d_x`

Fixed 16 dimensions:

```
[ log_len_instr, log_len_resp, log_total_len,
  resp/instr_ratio,
  loss_mean, loss_std, loss_max, perplexity,
  avg_logprob, entropy_mean, entropy_max,
  is_cot, is_code, is_qa, language_id_onehot(3 dims) ]
```

`loss/ppl/logprob/entropy` are obtained through one forward pass on the selector model.

### 7.3 Layer-Wise Activation Signature `a_x`

For each layer in `L_sig`, record:

```
[ ||h^attn_out||₂, ||h^mlp_out||₂, ||h^residual||₂,
  attn_entropy, attn_head_norm_var,
  mlp_activation_norm,
  hidden_token_var, last_token_dot_first ]
```

8 dimensions per layer, totaling 8 layers × 8 = 64 dimensions.

> Note: this shares the same forward pass as the site gradient signature, but the gradient signature additionally requires one backward pass. Cases without a loss mask may be skipped selectively.

## 8. Precise Construction of PEFT Representation `z_p`

### 8.1 Site Mask `m_p`

Shape: `|Ω| × 4` = 96 dimensions. The second dimension is the 4-bit indicator `[is_active, op_additive, op_multiplicative, op_prefix]`.

> Bottleneck adapter and additive low-rank are merged into `op_additive` in the operator dimension; prefix has a separate column; bias_shift is identified under `is_active=1, op_additive=0/mul=0/prefix=0` (one empty slot is reserved for future additions).

### 8.2 Capacity Vector `c_p`

```
[ log(trainable_params), trainable_ratio,
  lora_rank_normalized, lora_alpha_normalized,
  adapter_bottleneck_normalized, prefix_len_normalized,
  log_extra_flops, log_extra_memory_mb,
  inference_latency_ratio,
  affects_kv_cache,
  per_op_capacity_share (5 dims),
  per_op_indicator_count ]
```

16 dimensions.

### 8.3 Recipe Vector `r_p`

```
[ log(lr), warmup_ratio, weight_decay,
  scheduler_onehot(4),
  optimizer_onehot(3),
  log(batch_size), dropout,
  log(max_steps), grad_clip,
  init_method_onehot(2) ]
```

16 dimensions.

### 8.4 Functional Fingerprint `f_p` (Optional)

* Select a fingerprint probe set: 64 task-agnostic samples (short QA + short instructions).
* On `θ_base`, apply a small PEFT warmup once (h_fp=2 steps, fixed 8 samples), record `||Δh^ω||₂` at each site and downstream loss changes, and concatenate them into a 64-dimensional vector.

Optional module controlled by `PEFTConfig.use_fingerprint`; default `false`, enabled in ablation experiments.

## 9. Task Sketch Encoding

### 9.1 Sketch Construction Protocol

* Source: **only** the `train` or `dev` split of the target task, **never** the final `test` split;
* Default size: `N_V = 32`;
* Sampling: fixed seed, stratified into three layers by length quantiles (short/medium/long), sampled proportionally to avoid length bias;
* Persistence: `sketch_{task}_{seed}.json`.

### 9.2 Pooling

For the sketch sample set `V`, each sample obtains `z_v` with the same structure as `z_x`. Then:

```python
class TaskEncoder(nn.Module):
    # Set a latent query array Q ∈ R^{q × d}
    # MultiheadAttention(Q, K=V_set, V=V_set) → mean over q → z_t
```

Default `q = 4`, `d = 256`. Set-transformer-style attention pooling is more stable than mean pooling and more robust to long-tail sketch samples.

## 10. High-Fidelity Utility `u^hi`

### 10.1 Anchor Checkpoint Protocol

`A = 2`:

* `θ_base`: the original backbone, without any fine-tuning;
* `θ_warm`: obtained by randomly sampling 1k samples from the meta-pool and training a base-config LoRA (rank=8, attention-only) for 200 steps, serving as a representative "lightly fine-tuned" checkpoint.

Both anchors are frozen, and PEFT parameters are initialized on top of them.

### 10.2 Short-Horizon Update Protocol

For a tuple `(x, p, a, h)`:

1. Clone PEFT params from fresh init (per `p.init_method`);
2. Set optimizer = `p.recipe.optimizer` (default `adamw`), with empty state;
3. Repeat `h` times: run one forward + backward + step on `x`;
4. Run forward-only computation of `L_V` on `V`;
5. Return `Δ = L_V(θ_a) - L_V(θ_a + Adapt^h_p(x))`.

> Implementation note: all PEFT fine-tuning uses `torch.compile` or `fp16/bf16` acceleration; anchor model main parameters remain frozen throughout, so backward only affects PEFT parameters, saving about 50–90% backward time.

### 10.3 Noise Suppression

* In-bucket RankNorm: for all `Δ` values of `x` within the same `(p, t)` bucket, normalize by `rank/N` and convert to `[0,1]`.
* Weighted combination across horizons: default `w_1 = 0.4, w_4 = 0.6`.
* Default seed count is 1; when the scorer identifies uncertain regions, active sampling adds seeds instead of uniform repetition.

### 10.4 High-Fidelity Triplet Sampling

Total budget `Q_H = 10k`, divided into stages:

| Stage                 | Quantity | Strategy                                                                   |
| --------------------- | -------- | -------------------------------------------------------------------------- |
| Phase 1 (coverage)    | 50%      | Stratified sampling: sample cluster × PEFT family × PEFT capacity bucket   |
| Phase 2 (uncertainty) | 30%      | `q_query = σ̂ · (1 + γ · ReLU(û^lo))`, `γ=0.5`                             |
| Phase 3 (boundary)    | 20%      | Samples whose scorer ranking and true ranking disagree in the top-k region |

Phase 2 and 3 can only start after Phase 1 trains a v0 scorer.

## 11. Scorer Model

### 11.1 Inputs and Tensor Shapes

```
inputs:
  z_x  : (B, 848)
  z_p  : (B, 192) or (1, 192) broadcast
  z_t  : (B, 848) or (1, 848) broadcast
outputs:
  μ    : (B,)
  σ    : (B,)  via softplus, with floor=1e-3
```

### 11.2 Network Architecture

```
f_x:  z_x → LayerNorm → Linear(848, 256) → GELU → Linear(256, 256) → LayerNorm
f_p:  z_p → LayerNorm → Linear(192, 128) → GELU → Linear(128, 128) → LayerNorm
f_t:  z_t → LayerNorm → Linear(848, 256) → GELU → Linear(256, 256) → LayerNorm

# Conditional fusion: FiLM modulation + bilinear
cond = concat(h_p, h_t)                # 384
γ, β = Linear(384, 256) × 2
h    = γ ⊙ h_x + β                     # 256
bilin = Bilinear(h_x, h_p, 64)         # 64
out  = concat(h, bilin) → Linear(320, 256) → GELU → heads

μ_head: Linear(256, 1)
σ_head: Linear(256, 1) → softplus + 1e-3
```

Approximately 1.5M parameters; during deployment, single-sample forward ≈ 0.05ms (GPU).

### 11.3 Loss

```
L = λ_rank · L_rank^{hi}
  + λ_reg  · L_reg^{hi}
  + λ_proxy · L_proxy^{lo}
  + λ_unc  · L_unc^{hi}
```

Default weights: `λ_rank=1.0, λ_reg=0.3, λ_proxy=0.5, λ_unc=0.2`.

#### Pairwise ranking

Randomly sample pairs `(i, j)` within the same `(p, t)` bucket:

```
L_rank = - log σ( (μ̂_i - μ̂_j) · sign(u^hi_i - u^hi_j) )
```

#### Heteroscedastic NLL

```
L_unc = 0.5 · ((u^hi - μ̂)² / σ̂²) + 0.5 · log σ̂²
```

### 11.4 Training Plan

* **Stage A (low-fidelity pretraining)**: train the scorer only with `u^lo` labels for 3 epochs, AdamW lr=3e-4.
* **Stage B (joint fine-tuning)**: add `u^hi`, enable all losses, train for 2 epochs, lr=1e-4.

> Avoiding direct mixing prevents low-fidelity noise from dominating the initial training. First let the scorer learn the structure of z_x×z_p×z_t, then use sparse high-fidelity signals for correction.

## 12. Data Selection Strategy

### 12.1 Scoring

```
q(x) = μ̂(x, p*, t*) - λ · σ̂(x, p*, t*),  λ=0.2  (default)
```

### 12.2 Clustering

* Clustering space: `e_x.joint` (768 dimensions);
* Algorithm: MiniBatch K-Means;
* `k = max(50, ⌊√N⌋)`;
* For long-tail cluster sizes, apply `min_cluster_size = max(20, 0.1·B/k)` protection.

### 12.3 Quota Allocation

```
v_k = mean( top 10% of {q_i : x_i ∈ C_k} )    # cluster-level utility
b_k = round( B · (v_k^+)^α · |C_k|^{1-α} / Z )
α   = 0.6  (default; ablation α ∈ {0, 0.3, 0.6, 0.9, 1.0})
```

If `b_k > |C_k|`, clip to the upper bound, and redistribute the remaining budget to other clusters in proportion to `v_k^+`.

### 12.4 In-Cluster Selection

Select top-`b_k` according to q_i.

## 13. OOD Judgment and Calibration

### 13.1 ID Judgment

* After training, record `μ_z_p` and `Σ_z_p` for the PEFT configuration set used in training;
* ID score of a new PEFT `p*`:

```
d²(p*) = (z_{p*} - μ_z_p)^T Σ_z_p^{-1} (z_{p*} - μ_z_p)
```

* Threshold `τ_id = quantile(d²(p_train), 0.95)`.

### 13.2 Calibration Mode

If `d²(p*) > τ_id`:

1. Sample 200 / 500 examples (two comparison settings; see experimental design §6.3);
2. Compute high-fidelity `u^hi` for `p*` (horizon=1, single anchor, about 1/4 of the full cost);
3. Freeze the scorer body and train only a calibration head:

```
μ_cal = μ̂ + W_cal · [z_x; z_{p*}; z_t] + b_cal
```

4. Use these samples to fit `W_cal, b_cal` by linear regression;
5. Use `μ_cal` for full-pool scoring.

> **Implementation boundary (native short-horizon update backend)**: the high-fidelity short-horizon update in step 2 is implemented by `hi_fidelity.native_peft`, and only supports `lora / ia3 / adapter / bitfit` (`prefix / ptuning` require prompt/KV injection and cannot be represented as `nn.Linear` wrappers; see `native_peft.SUPPORTED_FAMILIES`). Therefore, **prefix / ptuning targets cannot generate calibration labels**. In E5, such L2 configurations can only use zero-shot, and the failure boundary should be reported honestly (experimental design §6.5); BitFit, which is also L2, can be calibrated normally. Calibration labels are generated by `scripts/experiments/build_calib_labels.py`.


### 16.2 Break-Even Computation Script

```python
def break_even_T(C_offline, C_apply, C_specific):
    return C_offline / max(1e-6, C_specific - C_apply)
```

The paper must render the above formula as a curve (X-axis: number of target PEFTs T; Y-axis: total cost).

## 18. Uncovered Items and Explicit Assumptions

The following items are **intentionally not detailed** in this design document and will be decided based on experimental results:

1. Final weights of λ_rank / λ_reg / λ_proxy / λ_unc — hyperparameter search;
2. Final value of `α` (cluster quota tilt) — determined by ablation;
3. Whether functional fingerprint enters the main results — determined by ablation;
4. Cost/benefit of increasing the number of anchors from 2 to 3 — follow-up experiments;
5. Whether to replace the selection space with DPP / submodular — backup option if adaptive cluster fails;
6. Extension across backbone families — explicitly left to future work.

## 20. Three Minimal Experiments for Validating Design Correctness (Runnable After M3)

1. **Correlation experiment**: sample 200 `(x, p, t)` triplets, compute `u^lo`, then compute high-fidelity `u^hi`, and compare Spearman ρ. If ρ < 0.3, the site definition or α formula needs revision.
2. **Necessity of PEFT condition**: train two scorers — with and without z_p input. Compare NDCG@K on hold-out triplets. The gap should be ≥ 5%.
3. **Equal-cost baseline comparison**: run this method and RDS+ / PPL / LESS under the same wall-time budget, then compare the post-training val loss of the selected subsets on the target PEFT. This method should at least match or outperform them, especially when multiple PEFTs share the scorer.

As long as these three experiments pass, the overall design loop is valid; subsequent work is engineering optimization and ablation experiments.
